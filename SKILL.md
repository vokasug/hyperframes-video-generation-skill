# hyperframes-video-generation

Генератор впечатляющих видео с озвучкой любой тематики: обучающие эксплейнеры, видео по книгам
(сюжет, смысл, мораль, выводы), обзоры, промо, отчёты, сторителлинг. Вход — файл (md/txt/pdf),
текст, ссылка или просто тема (фактура собирается из интернета). Выход — готовый MP4 с русской
(по умолчанию) или иной озвучкой, фоновой музыкой и современной моушн-графикой.

Скилл самодостаточен: включает TTS (edge-tts), шрифты, скрипты и проверенный на реальном
рендере контракт hyperframes. Скиллы hyperframes (`hyperframes-*`, `media-use`), если они
установлены на машине, подключаются как опциональные усилители — см. `references/enhancers.md`.

## Правило приоритета

Этот скилл — точка входа для ЛЮБОГО запроса на создание видео с озвучкой. Даже если на машине
установлен роутер `/hyperframes`, он НЕ используется как входная точка — только как библиотека
приёмов и блоков по решению этого воркфлоу. Исключение: пользователь явно просит другой скилл
по имени (например, «используй /faceless-explainer») — тогда это его сознательный выбор.

## Жёсткие правила

1. Сначала ПОЛНОСТЬЮ изучи источник (файл/текст/тему), выпиши термины, структуру, метафоры.
2. Вопросы пользователю — одним компактным блоком, с дефолтами (см. `references/questions.md`),
   пропуская то, что уже отвечено в запросе. Предлагай проработанные варианты, а не пустые поля.
3. Язык озвучки — русский по умолчанию. Если запрос пользователя не на русском — уточни язык
   и голос (edge-tts имеет десятки голосов: `edge-tts --list-voices`).
4. Три обязательные точки ревизии с подтверждением пользователя: ① бриф+раскадровка,
   ② снапшоты сцен, ③ draft-рендер. Не переходи дальше без «ок».
5. При конфликте инструкций побеждает контракт из `references/hyperframes.md` — он проверен
   реальным рендером. Усилители не меняют пайплайн, только обогащают.
6. Не удаляй и не изменяй файлы вне папки проекта. Чужие скиллы — только читать.
7. Готовое видео копируй в папку, указанную пользователем (по умолчанию — рядом с источником).

## Bootstrap (первый запуск или сомнение в окружении)

```bash
bash <skill_dir>/scripts/bootstrap.sh --check     # проверка окружения
bash <skill_dir>/scripts/bootstrap.sh --install   # глобальная установка недостающего
```

Ставит глобально (macOS Apple Silicon, Homebrew): node ≥ 22, ffmpeg, pipx + edge-tts,
кэширует `npx hyperframes`, опционально yt-dlp. Также сообщает, какие скиллы-усилители
hyperframes обнаружены на машине. Подробности: `references/pipeline.md`, этап 0.

Проект создаётся в `<cwd>/<slug>-video/` БЕЗ `hyperframes init` (скилл скаффолдит сам:
`package.json`, `index.html`, `meta.json`) — init автоматически ставит чужие скиллы глобально.
Если init всё же используется: `HYPERFRAMES_SKIP_SKILLS=1 npx hyperframes init ...`.

## Воркфлоу (8 этапов, 3 ревизии)

0. **Изучение источника** → фактура, термины, кандидаты в визуализацию, тип видео
   (нарративные шаблоны: `references/scripts.md`)
1. **Вопросы пользователю** одним блоком (`references/questions.md`)
2. **Бриф + раскадровка** (сцены 5–20 с, одна мысль = одна сцена, крупные подписи терминов)
   → **ревизия ①**
3. **Сценарий + TTS**: озвучка по сценам через edge-tts + субтитры SRT
   (`references/tts.md`, `scripts/tts_scenes.sh`; диалоги/подкаст — `references/podcast.md`,
   `scripts/podcast_tts.py`)
4. **Таймлайн**: SRT → старты/длительности сцен и метки предложений (`scripts/srt_timeline.py`)
5. **Сборка** `index.html`: сцены-архетипы (`references/scenes.md`), стиль
   (`references/styles.md`), локальные шрифты/GSAP, VO + фон (`references/music.md`)
   [усилители: hyperframes-animation, hyperframes-creative, hyperframes-registry]
6. **Валидация**: `npm run check` (0 ошибок) + снапшоты сцен → **ревизия ②**
   [усилитель: hyperframes-keyframes — диагностика движения]
7. **Рендер**: draft → **ревизия ③** → high (`npx hyperframes render -q high`)
8. **Проверка и выдача**: `scripts/verify_render.sh` (ffprobe, баланс голос/музыка,
   кадры из MP4) → копия результата пользователю

Полный регламент этапов и матрица деградации без усилителей — `references/pipeline.md`.

## Скрипты (scripts/)

| Скрипт | Назначение |
|---|---|
| `bootstrap.sh` | Проверка/установка окружения, детект усилителей |
| `probe_music.sh` | Доступность источников музыки с этой машины |
| `tts_scenes.sh` | Озвучка всех сцен из папки + SRT + замер длительностей |
| `podcast_tts.py` | Диалоги: per-line озвучка 2+ голосов, склейка сцен, lines.json меток реплик |
| `srt_timeline.py` | SRT → JSON таймлайна сцен и предложений |
| `install_fonts.sh` | Копия забандленных шрифтов в проект (+ fonts.css) |
| `fetch_fonts.py` | Fallback: скачать шрифты с Google Fonts |
| `prep_bgm.sh` | Обрезка фоновой музыки под длительность + фейды |
| `verify_render.sh` | Проверка готового MP4: формат, звук, кадры |

`<skill_dir>` = папка этого скилла (где лежит SKILL.md).
