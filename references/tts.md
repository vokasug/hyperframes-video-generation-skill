# Озвучка: edge-tts (бесплатно, без ключей)

Microsoft Edge TTS через `edge-tts` (pipx). Требуется интернет. Без API-ключей и аккаунтов.

## Голоса

- **Русский мужской (дефолт):** `ru-RU-DmitryNeural`
- **Русский женский:** `ru-RU-SvetlanaNeural` (есть также `ru-RU-DariyaNeural`)
- Другие языки: `edge-tts --list-voices | grep <xx-XX>` — если запрос пользователя не на
  русском, обязательно уточни язык и голос.

## Базовые команды

```bash
edge-tts --voice ru-RU-DmitryNeural --rate=+5% --file scene.txt \
  --write-media out.mp3 --write-subtitles out.srt
```

- Темп: дефолт `+5%` (бодро, но чётко); спокойный `0%`; максимум для обучалок `+10%`.
- Субтитры `.srt` генерируй ВСЕГДА — из них строится таймлайн (srt_timeline.py).
- Проверка результата: `ffprobe -v quiet -show_entries format=duration -of csv=p=0 out.mp3`.
- Пакетно: `scripts/tts_scenes.sh <dir_txt> <dir_out> [voice] [rate]`.

## Per-scene подход (обязателен)

Озвучивай КАЖДУЮ сцену отдельным файлом (`script/01-....txt` → `vo/01-....mp3`), а не одним
длинным: так сцены имеют жёсткие границы и не «уплывают», а SRT даёт метки предложений
для синхронизации анимаций внутри сцены.

**Структура предложений под синхронизацию**: если на сцене N элементов, которые должны
подсвечиваться по очереди (карточки, иконки, абзацы), — делай в VO ровно N отдельных
предложений-пунктов ПЛЮС отдельную вводную фразу. НЕ объединяй пункты списка через
точку с запятой в одно предложение («Реализуется по-разному: А; Б; В» — SRT даст один
кусок, и подсветка элементов станет невозможной). Пиши: «Реализуется по-разному. А. Б. В.»

## Диалоги / подкаст (несколько голосов)

`tts_scenes.sh` НЕ подходит — он озвучивает всю сцену одним голосом. Для диалогов:
`scripts/podcast_tts.py` (строки `S:`/`D:` в txt-сценах → per-line озвучка нужным голосом,
склейка с паузами 0.15с, lines.json меток реплик). Полный регламент — `references/podcast.md`.

## Подстановки произношения (пиши прямо в тексте)

**ОБЯЗАТЕЛЬНО читай `references/pronunciation.md`** — там полная система управления
ударениями и аббревиатурами (SSML не работает; ударения — дефисным разбиением
«эм-беддинг»; аббревиатуры — слитно «элэлэм» или через ZWJ «И‍И»; процесс кандидатов
для сомнительных слов). Короткая шпаргалка:

прослушиванием нельзя, поэтому подставляй заранее по таблице и здравому смыслу,
а сомнительные — через кандидаты (см. pronunciation.md).

## Попадание в длительность

- Плотность речи ≈ **2 слова/с** при +5% (2 мин ≈ 240–260 слов суммарно).
- Слишком длинно: 1) сократи текст (убери воду, вторые примеры), 2) темп до +8…+10%.
- Слишком коротко: расширь текст. Паузы между сценами >0.6 с не делай — провисает ритм.

## Субтитры в видео (опция)

Если пользователь просил полные субтитры: используй SRT (объедини сценовые со сдвигом на
старты сцен из timeline.json) и либо прожги в кадр (`ffmpeg -vf subtitles=...`), либо отдай
отдельным `.srt` рядом с MP4. Для караоке-подписей (подсветка слов) нужен word-level —
через транскрибацию из усилителя media-use (см. enhancers.md).
