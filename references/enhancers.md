# Усилители: скиллы hyperframes (опционально)

Наш скилл полностью автономен. Но если на машине установлены скиллы hyperframes
(лежат в `~/.config/opencode/skills/` или `~/.claude/skills/`), используй их как библиотеку
на фиксированных этапах. Их роутер `/hyperframes` НЕ является точкой входа — решения принимает
этот воркфлоу.

## Детект (на bootstrap и перед этапом 5)

```bash
ls ~/.config/opencode/skills/ ~/.claude/skills/ 2>/dev/null | grep -E "hyperframes|media-use"
```

Сообщи пользователю одной строкой: «усилители найдены: …» или «не найдены — идём автономно».

## Карта подключения

| Усилитель | Этап | Что даёт | Как использовать |
|---|---|---|---|
| `hyperframes-animation` | 5 (сборка) | библиотека приёмов: per-word typography, velocity transitions, audio-reactive, scene blueprints | загрузи скилл, бери 1–2 приёма на видео для разнообразия, НЕ заменяй архетипы из scenes.md полностью |
| `hyperframes-creative` | 2, 5 | креативные паттерны, beat planning, композиция | консультация при сомнениях в раскадровке |
| `hyperframes-registry` | 5 | 50+ готовых блоков: переходы (shader), чарты, оверлеи, субтитры | `npx hyperframes add <block>` — для 1–2 акцентных блоков (переход между актами, animated chart) |
| `hyperframes-keyframes` | 6 (валидация) | диагностика движения: мёртвые кадры, конфликты твинов | `npx hyperframes keyframes` после check; фикси найденное |
| `media-use` | 3, 8 | транскрибация (word-timings для караоке-субтитров), SFX, резолв медиа | только если нужны субтитры уровня слов или звуковые эффекты |
| `hyperframes-cli` | 7 | нестандартные команды (cloud/lambda render) | только по запросу пользователя |
| workflow-скиллы (music-to-video, embedded-captions и др.) | — | отдельные форматы | только по ЯВНОМУ запросу пользователя; предложи доустановить через роутер или обработай сам, если воркфлоу покрывает |

## Правила

1. **Приоритет ядра.** При любом расхождении с усилителями побеждает `hyperframes.md`
   этого скилла (проверенный контракт: data-*, GSAP-правила, id у audio, hard-kill).
2. **Точки фиксированы.** Усилители подключаются только на своих этапах из таблицы,
   пайплайн не перестраивают.
3. **Дозированно.** Не больше 1–2 внешних приёмов/блоков на видео — стиль должен остаться
   цельным (styles.md).
4. **Деградация.** Усилителя нет → делаем по матрице из pipeline.md, пользователя не дергать.
5. **Не трогать чужое.** Скиллы hyperframes только читаем; не редактируем, не удаляем,
   не переустанавливаем (кроме явной просьбы пользователя).
