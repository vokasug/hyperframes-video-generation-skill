# Подкаст-формат: видео с двумя (и более) голосами

Запрос вида «подкаст», «диалог», «два ведущих обсуждают». Отличие от монолога:
реплики чередуются по голосам, нужен визуальный индикатор говорящего и субтитры по репликам.
Проверено реальным рендером (книжный разбор «Идиота», 3:05, 36 реплик).

## 1. Сценарий с разметкой голосов

`script/<NN>-<slug>.txt` — те же сцены, но каждая строка = одна реплика с префиксом голоса:

```text
S: Как человек с самой чистой душой погубил всех, кого любил?
D: Это роман о князе, который оказался слишком хорош для своего мира.
```

- Одна реплика = 1–2 коротких предложения (≤15 слов) — динамика диалога держится на смене голоса.
- Роли задавай осмысленно: ведущий задаёт вопросы зрителя, эксперт отвечает (книжный разбор,
  интервью); или два равных в споре (дебаты). Дефолтная пара: ведущая `S` (Svetlana) +
  эксперт `D` (Dmitry). Три и более голосов — добавь префиксы в VOICES скрипта.

## 2. Озвучка

```bash
python3 <skill_dir>/scripts/podcast_tts.py . 0.15 "+5%"
python3 <skill_dir>/scripts/srt_timeline.py vo 0.25 0.30 > timeline.json
```

`podcast_tts.py` делает: per-line edge-tts нужным голосом → склейка сцены с паузами GAP
(0.15 с — живой диалог; больше 0.3 — вата) → `vo/<scene>.mp3/.srt` → `lines.json`.
Дальше глобальные метки реплик для аватаров/субтитров:

```python
# speakers.json = для каждой сцены timeline.json: start = scene.start + LEAD + lines.json[scene][i].start
```

Голоса и темп — в константах VOICES/RATE скрипта. НЕ используй tts_scenes.sh для диалогов —
он озвучивает всю сцену одним голосом.

## 3. Подкаст-панель (постоянный слой поверх сцен)

Один `<div class="clip">` на всю длительность, трек свой, DOM после сцен (z сверху).
Состав: стеклянная полоса снизу (100–120px), аватары слева, waveform, субтитры, прогресс-бар.

- **Контейнер**: дети панели — ТОЛЬКО внутри позиционированного родителя полосы
  (`bottom:0` у полосы не поможет, если дети позиционируются от `top` полноэкранного клипа —
  панель «уедет» наверх; см. troubleshooting).
- **Аватары**: SVG-портреты лайнартом в круге 64–68px (цвет голоса: она/он), рядом имя + роль
  (mono) и бейдж `ON AIR` (красная точка, пульс scale yoyo). Рисуй руками, 3–5 путей: голова,
  волосы/очки, плечи. Буквы вместо портретов выглядят дёшево — не используй.
- **Активный говорящий** (по меткам speakers.json, `tl.set` на каждую реплику):
  активный — `opacity:1, scale:1` + glow-кольцо (`box-shadow` дубликат, opacity 1) + ON AIR 1;
  неактивный — `opacity:.38, scale:.9`. Волну крась в цвет голоса через CSS-переменную:
  `tl.set("#wave", {"--wc": color}, t)`.
- **Waveform**: 5–9 полосок, `scaleY .25→1` yoyo `sine.inOut`, конечный repeat по формуле
  из hyperframes.md (repeat-overshoot!). Анимируется всегда, цвет меняется — эффект живой речи.

## 4. Субтитры по репликам (seek-safe)

НЕ твинь `textContent` — для каждой реплики свой div в одной точке:

```js
LINES.forEach(([sp, s, e, text]) => {
  const d = document.createElement("div");       // <span class="who">Имя</span> + текст
  subs.appendChild(d);                            // .who окрашен в цвет голоса
  tl.fromTo(d, {opacity:0, y:12}, {opacity:1, y:0, duration:.22}, s + 0.03);
  tl.to(d, {opacity:0, duration:.18}, e + 0.02);
  tl.set(d, {opacity:0}, e + 0.25);               // hard-kill
});
```

Шрифт 26–30px Manrope 600, зона по центру полосы, максимум 2 строки. Длинные реплики (>90
символов) режь ещё на этапе сценария.

## 5. Визуальная иерархия кадра

Подкаст-панель занимает низ (~120px) — сцены проектируй с безопасной зоной снизу:
контент сцен не ниже y≈950 при 1080p. Прогресс-бар 6px в самом низу, градиент стиля,
`scaleX 0→1` linear за всё видео.

## 6. Музыка

Тихая инструментальная (`data-volume` 0.12–0.14): под разговор не нужен ритм. Для книг/историй
проверена фортепианная классика (Сати — Gymnopédie No.1 с Wikimedia Commons, см. music.md).
