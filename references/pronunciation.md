# Управление произношением edge-tts (ударения, аббревиатуры, англицизмы)

Проверено в бою на ru-RU-DmitryNeural (действует и для остальных ru-голосов). Читать
ВСЕГДА перед генерацией озвучки: дефолтное чтение русских англицизмов и аббревиатур
часто неверно, а неправильное ударение в готовом видео — самый частый дефект.

## Главное: сервис НЕ принимает SSML

Бесплатный Edge Read-Aloud (его использует edge-tts) отклоняет любые SSML-теги —
`<phoneme>`, `<say-as>` и прочие возвращают `NoAudioReceived`. Управлять произношением
можно ТОЛЬКО написанием текста (подстановки прямо в txt-сценах).

| Приём | Работает? |
|---|---|
| Заглавная буква на ударном слоге (`тОкен`) | ❌ игнорируется |
| Удвоение гласной (`тоокен`) | ❌ игнорируется |
| Замена е/э (`токэн` вместо `токен`) | ⚠️ меняет звук, НЕ ударение |
| Знак ударения U+0301 после гласной (`языкова́я`) | ⚠️ избирательно: на «домашних» словах работает, на англицизмах игнорируется |
| **Дефисное разбиение (`эм-беддинг`)** | ✅ основной рабочий приём для ударений |
| **Слитное написание аббревиатур (`элэлэм`)** | ✅ чёткое чтение по буквам |
| **Zero-width joiner U+200D между буквами (`И‍И`)** | ✅ «и-и» без паузы |
| Пробел между буквами (`И И`) | ✅ то же, но с микропаузой |
| Дефис между буквами (`эл-эл-эм`) | ❌ смазывается, читается как одно слово |

## Правило 1. Ударения в англицизмах — дефисное разбиение

Дефис разбивает слово на просодические единицы, и модель естественно ставит ударение
на корневую часть. Проверенные пары:

| Слово | Дефолт TTS (неверно) | Рабочее написание |
|---|---|---|
| эмбеддинг | эмбеддИнг | **эм-беддинг** |
| эмбеддинги | эмбеддИнги | **эм-беддинги** |
| энкодеры | энкодЕры | **эн-кодеры** |
| энкодером | энкодЕром | **эн-кодером** |
| токен | токЕн | **токэн** (э-замена тут сработала; альтернатива — «то-кeн» не проверена) |
| языковые/-ых/-ой | языкОвые (или невнятно) | **язы-ковые / язы-ковых / язы-ковой** (дефис ПОСЛЕ корня «язык-», ударение само встаёт на -вы́х) |
| следующего слова | словА | **сло-ва** (не «сло́ва» — маркер U+0301 тут игнорируется) |
| каждого шага | каждого шуга | **ша-га** |
| студио (LM Studio) | студИо | **стьюдио** (мягкий знак форсирует стУдио; работает и «стуудио» — удвоение гласной) |
| языковой | языкОвой | **язы-ковой** (дефис ПОСЛЕ корня «язык-»; проверено: «языко-вой» в изоляции звучит ок, но в речи склоняется к «языкОвой» — всегда проверяй кандидата в ПОЛНОМ предложении сцены, а не изолированно) |
| многошагового | многошаговОго | **многоша-гового** |
| Gemini | геминИ | **ге-мини** |
| memory.md | меморИ эм дэ | **memory эм дэ** (само слово — латиницей, как user/soul; «ме́мори» с маркером тоже работает) |
| слова с долгим ударным гласным | — | техника: **удвоить гласную** («стуудио») или **поставить ь** («стьюдио») — оба способа подтверждены |

Для слов, где дефис не помогает, или «домашних» русских слов — пробуй U+0301
(после ударной гласной): «языкова́я» — сработало. Если не сработало — дополнительные
техники: **удвоение ударной гласной** («стуудио») или **мягкий знак** («стьюдио»),
а иначе — подбирай дефисное/иное написание через кандидаты (см. ниже).

## Правило 2. Аббревиатуры

- **GPT и подобные английские аббревиатуры — пиши латиницей как есть**: ru-голоса читают
  их корректно («GPT» → «джи-пи-ти» с правильным ударением). Кириллические подстановки
  («джи-пи-ти», «джипити́») дают неверное ударение — не использовать.
- **LLM** → `элэлэм` (слитно). Дефисный «эл-эл-эм» смазывается в одно слово.
- **ИИ** → `И‍И` (U+200D между буквами, zero-width joiner) → чистое «и-и».
  Вариант «И И» с пробелом тоже работает, но слышна микропауза. Без разделителя
  читается «имодель»/«иагентов» (буквы сливаются со следующим словом).
- **MCP** → `эм-си-пи`; **API** → `эй-пи-ай`; **RAG** → `рэг`; **LSP** → `эл-эс-пэ`;
  **MoE** → `эм-о-и` («МоЕ» читается как «моё»); **OWASP** → `овасп`;
  **LoRA** → `лора`; **fine-tuning** → `файн-тюнинг`; **AGENTS.md** → `агентс эм дэ`.
- Многословные: Claude Code → `Клод Код`, Codex → `Кодекс`, Cursor → `Курсор`,
  OpenCode → `Опенкод`, OpenClaw → `опен-кло` (дефис = ударение на «клО»), PicoClaw → `пико-кло`, Qwen → `квэн`, ZCode → `зет-код`,
  GLM → `джи-эл-эм`, Kimi → `кими`, Moonshot → `муншот`, Ollama → `оллама`,
  LM Studio → `ЛМ стьюдио`, llama.cpp → `лама цэ-пэ-пэ`.

## Правило 3. Процесс для ЛЮБОГО сомнительного слова

Угадать нельзя — проверяй кандидатами на слух. Это дёшево (2–3 файла × секунды).

1. Сгенерируй 2–4 написания-кандидата **внутри реального предложения сцены**
   (не изолированно — ударение зависит от контекста):
   `edge-tts --voice <голос> --rate=+5% --text "<фраза>" --write-media cand-A.mp3`
2. Опциональный пре-фильтр без прослушивания: огибающая энергии + высота тона
   по ядрам гласных (numpy, окно 30 мс): ударное ядро — длиннее/громче/выше.
   Интерпретируй осторожно — это только фильтр, финальное слово за слухом.
3. Отдай пользователю короткие файлы вида `pron/<слово>-A.mp3` с таблицей написаний
   и примени выбранное ко ВСЕМ сценариям (`grep -rn "<слово>" script/`).
4. Типовые ловушки, которые ловим так каждый раз:
   - «имодель»/«иагентов» — «ИИ» без разделителя сливается со следующим словом;
   - слияние согласных на стыке слов («чат-ботов и ИИ-агентов» — много «и» подряд,
     перефразируй: «чат-ботов, а также И‍И-агенты»);
   - буквы аббревиатур сливаются («эл-эл-эм» через дефис → «элэлэм» слитно);
   - следи и за грамматикой: TTS читает что написано — «лежат в основе кто? агенты»
     (номинатив), а не «агентов».

## Правило 4. Финальный контроль

Перед рендером: собери один контрольный mp3 со ВСЕМИ подозрительными словами проекта
в их реальных фразах и дай послушать пользователю (или предложи прослушать
draft-рендер с чек-листом слов). Исправление — повтор пунктов 1–3 + перегенерация
затронутых сцен и `timeline.json`.
