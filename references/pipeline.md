# Пайплайн производства видео

8 этапов + 3 обязательные точки ревизии. Порядок не нарушать: тайминги берутся из озвучки,
анимации — из раскадровки, раскадровка — из брифа.

## Этап 0. Окружение и изучение источника

**Bootstrap** (первый запуск): `scripts/bootstrap.sh --check`, при необходимости `--install`.

| Инструмент | Проверка | Установка (macOS, Apple Silicon) |
|---|---|---|
| Node.js ≥ 22 | `node --version` | `brew install node` |
| ffmpeg | `ffmpeg -version` | `brew install ffmpeg` |
| edge-tts | `edge-tts --version` | `brew install pipx && pipx install edge-tts` |
| hyperframes | `npx --yes hyperframes@latest --version` | не нужна глобальная, только кэш npx |
| yt-dlp (опц., музыка через YouTube при VPN) | `yt-dlp --version` | `brew install yt-dlp` |

**Изучение источника**: прочитай файл(ы) целиком. Выпиши: ключевые термины и определения,
естественную структуру (разделы → будущие сцены), метафоры и образы, числа и факты, кандидатов
в визуализацию. Если источник — только тема, собери фактуру из интернета (поиск/краулер) и
покажи пользователю краткую выжимку до брифа. Если источник — книга/статья: сюжет, герои,
ключевые идеи, мораль, выводы.

## Этап 1. Вопросы пользователю

См. `questions.md`. Одним блоком, с дефолтами, пропуская уже отвеченное. Для визуальных
решений предлагай 2–3 проработанных варианта на основе источника, а не открытые вопросы.

## Этап 2. Бриф + раскадровка → РЕВИЗИЯ ①

**Бриф** (коротко, для подтверждения): тип видео и нарративный шаблон (`scripts.md`), цель и
аудитория, длительность, язык/голос/темп, стиль-пресет (`styles.md`), формат кадра, музыка
(вариант + жанр), субтитры (да/нет), список сцен с метафорами.

**Раскадровка** (STORYBOARD.md в проекте): таблица по сценам —
`№ | длит. (оценка) | реплика озвучки | визуал | анимация | переход`.
Правила: сцена 5–20 с; одна мысль = одна сцена; на каждой сцене крупная подпись термина/тезиса;
8–14 сцен для 2 минут; последняя сцена — вывод/CTA.

Покажи бриф и раскадровку, дождись подтверждения. Не переходи к этапу 3 без «ок».

## Этап 3. Сценарий + озвучка

1. Напиши текст озвучки по сценам: `script/01-....txt`, `02-....txt`, ... — один файл на сцену,
   с подстановками произношения (`tts.md`). Ориентир объёма: ~2 слова/с при темпе +5%
   (2 мин ≈ 240–260 слов).
2. `scripts/tts_scenes.sh script assets/vo "<голос>" "+5%"` → mp3 + srt на сцену.
3. Проверь суммарную длительность: `Σmp3 + 0.55с × Nсцен` должна быть в целевой вилке.
   Длинно → сначала сокращай текст, потом темп до +10%. Коротко → расширяй текст, не растягивай
   паузы больше 0.6 с на сцену.

## Этап 4. Таймлайн

`scripts/srt_timeline.py assets/vo > timeline.json` — старт и длительность каждой сцены
(VO + lead-in 0.25 с + хвост 0.3 с) и абсолютные метки предложений для синхронизации
внутрисценовых анимаций. Эти числа — источник правды для `data-start`/`data-duration`
и позиций GSAP-твинов.

## Этап 5. Сборка композиции

1. Скаффолд проекта (БЕЗ `hyperframes init`): `package.json` с pinned-версией hyperframes,
   `index.html` по `hyperframes.md`. Папки: `assets/vo`, `assets/fonts`, `script`.
2. Шрифты: `scripts/install_fonts.sh .` (из бандла скилла) → `assets/fonts` + `assets/fonts.css`.
   GSAP: скачать `gsap.min.js` локально в `assets/` (никаких CDN при рендере).
3. Сцены по архетипам `scenes.md` в выбранном стиле `styles.md`. Один GSAP-таймлайн
   (paused) на всю композицию, абсолютные позиции из timeline.json.
4. Аудио: VO-клипы (каждый с уникальным `id`!) + `bgm.mp3` (`prep_bgm.sh`, `data-volume`
   0.12–0.15). Контрольный список контракта — `hyperframes.md`.
5. [Усилители, если есть]: приёмы из hyperframes-animation/creative, блоки из
   hyperframes-registry (`npx hyperframes add <block>`). Правила — `enhancers.md`.

## Этап 6. Валидация → РЕВИЗИЯ ②

```bash
npm run check                                        # 0 ошибок обязательно
npx hyperframes snapshot --at <середины сцен и пики> # PNG для ревизии
```

Снапшоты: середина КАЖДОЙ сцены + пики экшена (момент штампа, блокировки, заполнения
контекста) + стыки сцен. Смотри: переполнения текста, пустые кадры, видимые «хвосты»
прошлой сцены, читаемость подписей. [Усилитель]: `npx hyperframes keyframes` — диагностика
движения. Покажи контакт-лист пользователю, дождись «ок».

## Этап 7. Рендер → РЕВИЗИЯ ③

```bash
npx hyperframes render -q draft -o renders/v1-draft.mp4   # быстрая ревизия
# ... правки при необходимости ...
npx hyperframes render -q high -o renders/v1.mp4          # финал
```

Ориентир времени high-рендера: ~2–2.5× длительности видео (2 мин ≈ 4.5 мин на M-серии).
Версии не затирай: v1, v2, ...

## Этап 8. Проверка и выдача

`scripts/verify_render.sh renders/v1.mp4` — формат/длительность/дорожки, громкость речи
(mean ≈ −20 dB), музыка на 13–17 dB ниже, кадры из готового MP4 (не только снапшоты!).
Скопируй итог в папку пользователя с понятным именем, сообщи путь, длительность и размер.

## Матрица деградации (усилители отсутствуют)

| Нет усилителя | Делаем так |
|---|---|
| hyperframes-keyframes | check + снапшоты + глазами по кадрам MP4 |
| hyperframes-registry | переходы/блоки руками по scenes.md (fade/scale/slide) |
| hyperframes-animation/creative | приёмы из scenes.md + styles.md (их хватает) |
| media-use | субтитры из наших SRT (ffmpeg burn-in или отдельным файлом) |
